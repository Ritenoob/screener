## ðŸ“ Notes for Designers

### Critical Design Considerations

1. **Financial Data Display**
   - Always use monospace/tabular figures for numbers
   - Right-align numerical data in tables
   - Show + or - sign explicitly for P&L
   - Use consistent decimal places (2 for USD, 8 for crypto)

2. **Color for Meaning**
   - Green = profit, bullish, success, safe
   - Red = loss, bearish, danger, risk
   - Never use color as the only indicator
   - Include icons, text, or patterns

3. **Real-Time Feel**
   - Subtle animations for value changes
   - Pulse indicators for active/live states
   - Timestamp everything
   - Show connection status prominently

4. **Information Hierarchy**
   - Critical info (balance, P&L, alerts) always visible
   - Progressive disclosure for details
   - Clear visual grouping
   - Consistent placement across screens

5. **Error States**
   - Design all error scenarios
   - Provide clear recovery actions
   - Don't hide errors
   - Make error messages actionable
